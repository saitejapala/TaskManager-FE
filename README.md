Kinetic is a professional, high-performance Task & Work Item management platform. It features a modern flat design, theme synchronization (dark/light), and clean vertical-slice architecture to support robust modular scaling.

---

## 🚀 Key Features

* **Vertical-Slice Architecture**: Code is split by feature slices (`auth`, `workItems`), grouping services, slices, schemas, hooks, pages, and components together rather than dumping them flat.
* **OTP-Integrated Signup Flow**: Implements a secure two-step registration flow. Registration details from the signup page are carried in the router state and processed with verification codes in the OTP screen.
* **Global Theme Syncing**: Implements responsive dark/light mode toggles powered by a global React Context provider and native CSS variable lookups.
* **Self-Decoding JWT Tokens**: The Redux auth slice decodes standard Dotnet JWT claims to load name and email properties dynamically for profile initials rendering.
* **Work Item CRUD**: Complete dashboard interfaces to create, read, update, and delete tasks inside standard modals. Includes interactive completeness toggles.
* **Local Search Filtering**: Real-time task searches throttled via a custom `useDebounce` hook.
* **Reusable Validation Rules**: Shared schema constraints (e.g. `emailValidation`) and generic validation functions (like `matchFieldsRefinement` to verify multiple matching confirm fields) inside a shared validations folder.

* **Core**: React 18 + TypeScript (Strict Mode)
* **Build Tooling**: Vite 5
* **State Management**: Redux Toolkit + Redux Persist (Whitelists authentication state in localStorage)

* **Form & Validation**: React Hook Form + Zod
* **Styling**: Tailwind CSS v3 (Flat design presets, Space Grotesk headings, DM Sans body fonts)

## ⚙️ Getting Started

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed (v18+ recommended).

### 1. Install Dependencies
```bash
npm install
```

### 2. Configure Environment Variables
Create a `.env` file in the root directory (based on `.env.example`):
```env
VITE_API_BASE_URL=http://localhost:5033
```

### 3. Run Development Server
```bash
npm run dev
```
The application will open at `http://localhost:3000`.

### 4. Build for Production
To build the optimized static asset bundle:
```bash
npm run build
```

### 5. Code Quality Check
To check and enforce type safety and linter constraints:
```bash
# Typecheck
npx tsc --noEmit

# Linter
npm run lint
```
